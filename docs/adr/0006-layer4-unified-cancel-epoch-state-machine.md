# ADR 0006 — Layer-4 unifies the async-halt cancel/epoch protocol into one modelled state machine

**Status:** Accepted (2026-05-27) · **Issue:** #10 · **Affects:** Layer 2/4 (`coordination/admission.py` — `note_execution_transition`, `cancel_ssh_tier`, `complete`, `close_admission`, `_is_stale_lifecycle_close`), Layer 4 (`server.py` `target.run_tests` rewire, the stop-capable controller)

## Context

The execution-state gate's async-halt cancellation accreted across the Layer-1+2 adversarial-review loop as a **cluster of stacked timing-edge fixes**, each exposing the next (review rounds r2→r5→r7→r8→r9→r10):

- r2 — generation-fence the epoch bump (`note_execution_transition`).
- r5 — fence the cancel by halt-epoch.
- r7 — don't drop the whole cancel; filter **per handle** by admitted epoch (`admit_epoch <= halt_epoch`).
- r8 — generation-fence the lifecycle close.
- r9 — also skip the teardown emit on a stale lifecycle retry.
- r10 — fence `complete()` by epoch as a backstop for the cancel-delivery gap.

Today the invariant is spread across `note_execution_transition` (the sole epoch bumper, generation-fenced), `cancel_ssh_tier` (generation fence **plus** a per-handle `admit_epoch <= halt_epoch` filter), `complete` (an `admit_epoch != current_epoch` backstop for the record-vs-deliver window), and `close_admission`/`_is_stale_lifecycle_close`. Each is individually defensible and all held through 10 rounds without reverting — but they are *refinements of prior fixes*, which is the documented signal of fixing symptoms rather than the mechanism. Layer 4 is the first layer that drives this gate against the **real** `target.run_tests` path (killing an in-flight ssh `subprocess` on halt rather than letting it block under the tests lock), so it is the right and necessary point to consolidate before adding a second consumer (the future `debug.introspect` tier) on top of the patch stack.

Adversarial review cannot settle this: a concurrency protocol's interleavings are combinatorial and a fresh reviewer re-litigates each round. Tests — specifically an interleaving/property pass — are what falsify a concurrency design cheaply.

## Decision

- **One authoritative model.** Define the per-`TargetKey` execution-gating state as a single `(generation, execution_epoch, execution_state)` machine with an explicit, enumerated transition table. `generation` is the incarnation (bumped by provisioning lifecycle), `execution_epoch` bumps on **every** stop-capable execution-state transition, `execution_state ∈ {EXECUTING, HALTED, UNKNOWN}`. The table is written into the plan as transitions × fences, not prose.
- **Replace the patch stack with the model.** `note_execution_transition`, `cancel_ssh_tier`, `complete`'s epoch backstop, and the lifecycle-close fences are re-expressed as the transition function and a small set of named **fences** (`generation_fence`, `epoch_fence`, `guard_token_authority`) applied uniformly, rather than as six independently-reasoned guards. Where the unified model proves an existing guard redundant, the guard is removed (not left as defensive duplication); where it proves one necessary, it is the same fence applied at that transition.
- **Authority stays the guard token (ADR 0002).** The model does not introduce a parallel stop-session authority. The `generation` fence rejects prior-incarnation events; the **live `StopCapableGuard` token** remains the sole authority distinguishing a stale *same-generation* controller (a detached session's late worker), exactly as today. This ADR reorganizes the fences; it does not move authority.
- **Cancel delivery to the in-flight op is a teardown-bounded watcher bridge (amendment 2026-05-27, review Finding #2; teardown corrected in the second review round, Finding #1).** The transition table fences *admission*, but a halt must also physically **reach** the live ssh subprocess — the model and the runner hold different objects (the handle's `_cancel` fence vs. the runner's `threading.Event`). The delivery mechanism: after `admit_ssh_tier` returns an `AdmissionHandle`, the `target.run_tests` path spawns a daemon watcher that **polls** `handle.wait_cancelled(0.1)` inside a loop guarded by a `watch_done` `Event`, and the handler **`join`s the watcher in a `finally`**. It is specifically *not* the fire-and-forget `if handle.wait_cancelled(): runner_cancel.set()` form: `wait_cancelled(timeout=None)` parks until `_cancel` is set, but `complete()`/`_dispose_locked()` on the normal completion path never set `_cancel` (only the cancel paths do — `coordination/admission.py:671,774` vs. `:648,819`), so an unbounded watcher would block forever on every successful run and leak one daemon thread + the pinned completed handle + the closure per `target.run_tests`. The halt path (`note_execution_transition` + `cancel_ssh_tier`) sets the handle's private `_cancel` fence; the bounded watcher observes it within one poll interval and sets the runner's `cancel` event, which kills the in-flight `Popen` (plan Tasks B1/B2). No separate cancel registry is needed — `cancel_ssh_tier` already fences every live `SSH_TIER` binding for the target; the watcher is the per-op delivery edge from that fence to the subprocess. **Invariant:** the `watch_done`/`join` teardown runs on the success, cancel, and `execution_state_changed` exit paths alike, so a normal (non-halted) completion always tears the watcher down — no parked thread, no pinned `AdmissionHandle`. And when `complete()` raises `execution_state_changed` (the op spanned a halt), the handler **terminalizes the RUNNING manifest `StepResult` to FAILED** and returns failure — it never leaves a stale RUNNING step nor records SUCCEEDED.
- **Tests adjudicate, not review rounds.** The protocol ships with a **hypothesis stateful test** (or, if the state space proves too irregular for a clean `RuleBasedStateMachine`, a hand-written interleaving matrix) that drives the transition table — admit/halt/resume/cancel/complete/close/invalidate in adversarial orders — and asserts the invariants: no stale `EXECUTING` survives a halt; an ssh op spanning a halt rolls back, never completes; a delayed cancel still cancels its pre-halt op without touching post-resume work; a prior-incarnation event is a no-op. The pass covers the SSH_TIER admit + cancel + complete + lifecycle path; the TRANSPORT_OPEN admit/close path is covered via dedicated rules (`admit_transport_open`, `close_transport_open`, `mark_recovery_required`) that exercise the recovery-gated admit edge (§4.7) of the same model — driving the dual-write tombstone+cache invariant and asserting that a current tombstone gates BOTH non-recovery transport.open AND admit_ssh_tier (the two sides of the same §4.7 fence). Other tiers (e.g. a future `debug.introspect`) are not modeled. Merge bar for the gate is **finding quality + a green property pass**, not reaching `approve` in the review loop.

## Consequences

- The async-halt edge cluster becomes one reviewable artifact with one rationale, so a future change reasons about the transition table rather than re-deriving six interacting guards. The next consumer (`debug.introspect`) reuses the model instead of re-discovering its edges.
- There is real risk of **behavioral regression** while replacing guards that currently pass. Mitigation: the existing Layer-2 conformance tests for each edge (the r2/r5/r7/r8/r9/r10 cases) are kept and must stay green under the unified implementation — the model must *subsume* them, proven by re-running them, before any are deleted as redundant.
- The property/interleaving pass is new test infrastructure (a `hypothesis` dev-dependency if not already present). That cost is the point: per the project's own retro, running tests is the only thing that falsified a self-contradictory concurrency design that ~17 review rounds missed.
- If the unified model reveals that one of the stacked fixes was compensating for a different latent bug, that surfaces as a property-test counterexample during Layer 4 rather than in production — the intended outcome.
- **Production lifecycle producer in #10 is the reaper (amendment 2026-05-27, post-implementation review Finding #3).** `registry.reconcile()`'s orphan-backend reap calls `admission.invalidate_lifecycle(target_key, CRASHED)` via an injected callback (`SessionRegistry(on_orphan_reaped=...)`), so the §4.5 chain (lifecycle event → admission invalidates → `_SessionSubscriber.force_drop` → guard/lease release + record delete + handle deregister) is exercisable end-to-end without test scaffolding. The closure body lives in `server._build_transport_machinery`, keeping `coordination/registry.py` free of admission/lifecycle imports. Other producers (libvirt domain stop, qemu-gdbstub disconnect, agent-proxy heartbeat) are deferred to their owning layers; #10 ships exactly one production source of `LifecycleEvent`.
- **`inject_break` success path post-probes execution state (amendment 2026-05-27, post-implementation review Finding #4).** The mechanism's return value is no longer authoritative; after `break_mechanism(...)` returns successfully, the handler re-probes the execution state. **Round-2 correction (Findings F2/F5):** the post-probe is `probe_rsp_halted` — a real bounded RSP `?` exchange against the session's `rsp_endpoint`, returning True iff the peer answers with a stop reply (`T..`/`S..`) — NOT `probe_execution_state`. The cached-flag read would have been circular: `_halt_debug_transport` writes `HALTED` to the very flag the probe would read, so `break_unconfirmed` was unreachable on the success path against a silent-no-op break mechanism. On `probe_halted == False` — including a probe exception/timeout — the handler dual-writes UNKNOWN to the durable record and returns `DEBUG_ATTACH_FAILURE/break_unconfirmed`, preserving the existing exception-branch posture and the §5.6 "no optimistic admit" rule. The cached-flag read remains the ssh-tier admit fact (`admit_ssh_tier` / `probe_execution_state`), per [ADR 0001](0001-layer2-layer4-execution-state-gate-split.md)'s amended "Two distinct probes" table; the two probes have different consumers and different freshness guarantees. **`end_session(force=True)` remains a caller-asserted contract in #10:** the caller declares the kernel is resumed and accepts that a misuse leaves the next attach without `recovery_required` gating against a still-halted target. Adding a post-probe to that path is deferred — its semantics (probe→HALTED ⇒ override the force suppression and write a tombstone) require a separate review.
- **Lifecycle close on orphan reap is conditional (round-2 amendment, Finding F1).** `SessionRegistry.reconcile()` distinguishes (a) a live orphan whose backend `stop_by_identity()` actually killed — `close_admission=True`, the §4.5 cancel-fence + emit chain runs — from (b) a record whose backend was already dead, fingerprint-unverifiable, or `backend_pid is None` (the qemu-gdbstub case) — `close_admission=False`, only the lifecycle event is emitted, no `_closed_at` write. Before the fix, every reap closed admission; no production code path calls `reopen()`, so a single surviving durable record from a prior server lifetime permanently bricked admission for that target until process restart. `OrphanReap.close_admission_required` carries the bit; `SessionRegistry.reconcile()` returns an `OrphanReapReport` listing reaps and callback failures (formerly silently swallowed by `contextlib.suppress` — Finding F9), surfaced through the project logger by `_build_transport_machinery`.

## Considered & rejected

1. **Carry the Layer-2 protocol forward unchanged; only extend for the real `run_tests` kill-on-halt.** Lowest churn; the stack held 10 rounds. **Rejected:** it adds a second live consumer (and later a third, `debug.introspect`) on top of a mechanism whose own retro flags it as symptom-patching; the r5→r7 and r8→r9 pairs are refinements-of-fixes, the documented signal that the mechanism — not just its edges — needs one coherent statement. Deferring the consolidation only raises the cost of doing it.
2. **More adversarial-review rounds on the protocol design before coding.** **Rejected:** a concurrency protocol is unfalsifiable in prose review — combinatorial interleavings, fresh-reviewer re-litigation, oscillation. The project's recorded experience is that review consumed the full round cap on this exact surface without converging, and that a one-token test fixture exposed a contradiction the rounds never did. Adjudication moves to tests.
3. **Model-check the state machine (TLA+/Alloy) instead of property tests.** Strongest interleaving coverage. **Rejected for now:** introduces a toolchain and modelling language outside the repo's Python/pytest idiom for a state space small enough that `hypothesis` stateful testing covers it; revisit only if the property pass proves inadequate. Recorded so it is not relitigated.
